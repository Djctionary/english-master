# v0.5 — Light/Dark Theme Switch

**Date:** 10/04/2026

## Goal

Add a light/dark theme toggle to the app. Leverage the existing CSS custom-property design system from v0.4 so switching themes only requires swapping token values on `<html>`.

---

## Design

### Behavior

1. **Default:** follow system preference (`prefers-color-scheme`).
2. **Toggle:** sun/moon icon button in the top-right corner of every page.
3. **Persistence:** save choice to `localStorage` (`theme` key). On load, apply saved preference first; fall back to system preference.
4. **No flash:** inject a blocking `<script>` in `<head>` to read the stored theme before first paint.

## Implementation Plan

### 1. Dark tokens in `globals.css`
Add `[data-theme="dark"]` selector that overrides all `--color-*` and `--shadow-*` variables.

### 2. Anti-flash script in `layout.tsx`
Add a blocking inline `<script>` inside `<head>` that reads `localStorage.getItem("theme")` and sets `data-theme` on `<html>` before React hydrates.

### 3. `ThemeToggle` component
- Client component with sun/moon icon.
- Reads/writes `data-theme` attribute and `localStorage`.
- Listens for system `prefers-color-scheme` changes when no stored preference.

### 4. Place toggle in layout
- Add `ThemeToggle` to `app/layout.tsx` body, positioned fixed top-right.
- Ensure it's accessible on all pages (learn, review).

### 5. Fix remaining hardcoded colors
- `CorrectionComparison.tsx` — replace 3 hardcoded hex values with CSS variables.
- `ConfirmDialog.tsx` — replace overlay rgba with CSS variable.

- Update `CHANGELOG.md`
