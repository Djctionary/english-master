# v1.0-alpha — User Management + Landing Page

**Date:** 10/04/2026

## Goal

Transform English Master from a single-user tool into a multi-user application with authentication, per-user data isolation, and a public landing page that communicates the product vision.

---

## Positioning

**For:** English learners who already have the basics — and are hungry for more vocabulary, pronunciation, and natural expression.

**Core insight:** A sentence is a capsule of a real moment. Capture it, and you capture the context, the emotion, the meaning. Review it later, and you don't just recall words — you relive the experience. That's how language becomes part of you.

**Slogan:** "Sentence is all you need."

---

## Requirements

### 1. Authentication

- Username + password registration and login (no email required)
- Password: minimum 4 characters, stored as bcrypt hash
- Session: JWT in httpOnly cookie, 7-day expiry
- Logout button in nav (learn + review pages)
- No email validation, no password recovery (MVP)

### 2. Per-user Data Isolation

- `users` table: id, username, password_hash, created_at
- `sentences` table: add `user_id` column (FK → users)
- `sentence_review_states` table: add `user_id` column (FK → users)
- All queries filter by authenticated user's ID
- Migration: create user "vergil" (password: "yamato"), assign all existing data

### 3. Registration Cap

- Maximum 100 registered users
- Registration endpoint checks user count, returns 403 if >= 100
- Landing page shows remaining spots counter (e.g., "93 spots remaining")

### 4. Route Protection

| Route | Access | Behavior |
|---|---|---|
| `/` | Public | Landing page with login/register |
| `/learn` | Protected | Redirect to `/` if unauthenticated |
| `/review` | Protected | Redirect to `/` if unauthenticated |
| `/api/auth/*` | Public | Register, login, logout, me |
| `/api/*` (all others) | Protected | 401 if no valid JWT, filter by user_id |

Implementation: Next.js `middleware.ts` checks JWT cookie.

### 5. Landing Page (`/`)

**Layout:**

1. **Hero** — "Sentence is all you need." + positioning tagline + [Start Learning] CTA
2. **Why Sentences?** — Comparison table (Traditional Apps vs English Master)
3. **How It Works** — 4-step flow: Capture → Correct → Tag → Review
4. **Alpha badge** — "V1.0 Alpha · Internal Test · First 100 users enjoy lifetime membership when v1.0 launches."
5. **Auth overlay** — Modal dialog triggered by [Start Learning], with Login/Register tab switch

**Advantage table:**

| | Traditional Apps | English Master |
|---|---|---|
| Learning unit | Single word / flashcard | Full sentence with real context |
| Context | Dictionary definition | You know where you found it — a movie, a game, a conversation |
| Input | Passive — read & memorize | Active — type what you want to say, AI corrects you |
| Review trigger | Abstract repetition | Reliving the moment you captured the sentence |
| Retention | Forget after exam | Sticks forever — your sentence, your memory |

**How It Works:**

1. **Capture** (10 seconds) — See something interesting. Type it — even with mistakes.
2. **Correct & Analyze** — AI corrects your sentence, breaks down grammar, vocabulary, pronunciation.
3. **Tag & Collect** — Tag by source. Build your personal sentence library.
4. **Review & Remember** — Spaced repetition brings it back. Hear the audio, recall the moment.

**Design:** Same "Calm Education" design system, supports light/dark mode.

### 6. Username Display + Encouragement

- Header in learn/review pages shows: `Hi, {username}` alongside nav buttons
- After completing 10 reviews in a session, show encouragement:
  "Great work, {username}! 10 sentences reviewed — consistency builds fluency."

### 7. Future (not in this version)

- Multilingual background art ("Learn English from Sentences" translated into many languages)
- Email validation, password recovery
- Payment / subscription system
- Lifetime member system implementation

## Database Schema

```sql
CREATE TABLE IF NOT EXISTS users (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  username      TEXT UNIQUE NOT NULL,
  password_hash TEXT NOT NULL,
  created_at    TEXT NOT NULL DEFAULT (datetime('now'))
);

-- Migration for existing tables
ALTER TABLE sentences ADD COLUMN user_id INTEGER REFERENCES users(id);
ALTER TABLE sentence_review_states ADD COLUMN user_id INTEGER REFERENCES users(id);
```

## API Endpoints

### Auth

```
POST /api/auth/register  { username, password } → { user } | 400 | 403 (cap)
POST /api/auth/login     { username, password } → { user } | 401
POST /api/auth/logout    → 200 (clears cookie)
GET  /api/auth/me        → { user } | 401
```

### Existing (now require auth)

All existing endpoints unchanged in interface, but:
- Read user_id from JWT cookie
- Filter all queries by user_id
- Insert new records with user_id
