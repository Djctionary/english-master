# v1.2-alpha — Audio Persistence Fix + TTS Upgrade

Critical fix: audio was being silently regenerated on nearly every playback
on Vercel because MP3 files lived on the ephemeral `/tmp` filesystem. This
version moves audio binaries into the database so they survive cold starts,
and upgrades the TTS model/voice.

## The Bug

- `generateAudio()` wrote MP3 to `getAudioDir()` (→ `/tmp/data/audio/` on Vercel).
- Only the **filename** was stored in Postgres.
- `/tmp` is wiped on every Lambda cold start — files never survive between invocations.
- `/api/audio/[filename]` had a fallback that regenerated via ElevenLabs and wrote
  back to `/tmp` (which got wiped again).
- Net effect: every audio playback after a cold start was a paid ElevenLabs call.

## Changes

### 1. TTS Upgrade
- Model: `eleven_turbo_v2_5` → **`eleven_multilingual_v2`**.
- Default voice: `EXAVITQu4vr4xnSDxMaL` ("Sarah") → **`DXFkLCBUTmvXpp2QwZjA`**.
- New env overrides: `ELEVENLABS_VOICE_ID`, `ELEVENLABS_MODEL_ID`.

### 2. Audio Persistence — store binary in the database
- New column `audio_data` (BLOB on SQLite, BYTEA on Postgres) on `sentences`.
- `generateAudio()` is now a **pure function** — returns `{ filename, data: Buffer }`
  with no filesystem writes.
- `/api/analyze` persists both `audio_filename` **and** `audio_data` when the sentence
  record is inserted or re-analyzed.
- `/api/audio/[filename]` serves the binary **directly from the DB**.
- Legacy records that have a filename but no `audio_data`: regenerate once via
  ElevenLabs, persist to the column, then serve. One-time cost per sentence.
- Dropped the ephemeral `/tmp` writes entirely — DB is the single source of truth.
- Added `Cache-Control: public, max-age=31536000, immutable` — filenames are
  content-addressed (SHA-256 of the sentence) so CDN/browser caching is safe.

### 3. Removed dead code
- `getSentenceByAudioFilename()` from both `lib/db.ts` and `lib/sentence-store.ts`
  (only consumer was the old filesystem-fallback path).

### 4. Tests updated
- `lib/openai.test.ts`: rewrote `generateAudio` suite to assert on the
  `{ filename, data }` return shape and confirm the new model/voice defaults.
- `api/audio-filename.test.ts`: rewrote to drive a SQLite in-memory DB,
  verifying (a) direct BLOB serving, (b) lazy regeneration + persistence when
  `audio_data` is NULL, (c) 404/502 paths.
- `api/analyze.test.ts`: updated `generateAudio` mocks to the new shape.

## Migration

- SQLite: `ALTER TABLE sentences ADD COLUMN audio_data BLOB` runs on next boot.
- Postgres: `ALTER TABLE sentences ADD COLUMN IF NOT EXISTS audio_data BYTEA`.
- Existing rows keep their `audio_filename` but have `audio_data = NULL` until
  first playback (lazy backfill regenerates once and caches forever).

## Env vars

- `ELEVENLABS_VOICE_ID` → set to `DXFkLCBUTmvXpp2QwZjA` in `.env.local` (and Vercel).
- `ELEVENLABS_MODEL_ID` → defaults to `eleven_multilingual_v2`; override if you want
  to revert to turbo.
