## Data Store Provider

The app now supports one storage API with two backends:

- `sqlite` for local development
- `postgres` for persistent production storage (recommended on Vercel)

Selection logic:

1. If `DATASTORE_PROVIDER` is set to `sqlite` or `postgres`, that value is used.
2. Otherwise, if `DATABASE_URL` is set, Postgres is used.
3. Otherwise, SQLite is used.

### Local development (SQLite)

No extra setup is required; data is stored in `./data/sentences.db`.

### Vercel production (Postgres)

Set a `DATABASE_URL` environment variable that points to your Vercel Postgres instance.
The app will auto-create the `sentences` table on first request.
